# 运营数据看板

**文档版本：** v2.1 Phase 3
**更新日期：** 2026-05-08
**适用对象：** 运营人员、产品经理

---

## 一、概述

静态博客项目受限于技术架构，无法实现自动化的实时数据看板。本文档提供手动创建和查看数据看板的方案，以及推荐的看板结构。

## 三、GA4 数据看板配置

### 3.1 创建自定义仪表盘

1. 登录 [Google Analytics](https://analytics.google.com/)
2. 进入「报告」→「库」
3. 点击「新建自定义仪表盘」

### 3.2 推荐组件配置

#### 核心指标卡片

| 指标 | 指标类型 | 显示格式 |
|------|----------|----------|
| 活跃用户 | `totalUsers` | 数字 + 趋势箭头 |
| 会话 | `sessions` | 数字 |
| 页面浏览量 | `screenPageViews` | 数字 |
| 平均参与时长 | `averageSessionDuration` | 分:秒 格式 |

#### 趋势图组件

```
报告类型：时序图
维度：日期
指标：页面浏览量、用户数、会话数
时间段：最近 30 天
```

#### TOP 10 文章列表

```
报告类型：数据表
维度：页面路径
指标：页面浏览量、平均参与时长、跳出率
排序：页面浏览量（降序）
过滤：页面路径包含 /20XX/
```

### 3.3 分享仪表盘

1. 在仪表盘右上角点击「分享」
2. 选择「发布到生态系统」或「导出为 PDF」
3. 设置访问权限

## 四、数据查看入口

### 4.1 Google Analytics 4

| 报告类型 | 访问路径 | 用途 |
|----------|----------|------|
| 实时 | 报告 → 实时 | 监控当前访客 |
| 用户获取 | 报告 → 用户获取 | 流量来源分析 |
| 参与度 | 报告 → 参与度 | 用户行为分析 |
| 转化 | 报告 → 转化 | 目标达成情况 |

### 4.2 快速访问模板

#### 每日检查模板

```
1. 打开 GA4 → 实时报告
2. 查看当前在线人数
3. 查看今日核心指标
4. 检查异常流量
```

#### 周报数据模板

```
1. 用户获取 → 概览（本周 vs 上周）
2. 参与度 → 页面和屏幕（TOP 10 文章）
3. 参与度 → 事件（关键事件触发次数）
4. 转化 → 漏斗（各阶段转化率）
```

## 六、自定义报告模板

### 6.1 周报数据提取路径

#### 1. 流量概览

- 路径：报告 → 概览
- 提取指标：用户数、页面浏览量、会话数、平均参与时长

#### 2. 流量来源

- 路径：报告 → 用户获取 → 流量获取
- 提取维度：会话来源、媒介
- 提取指标：用户数、转化数

#### 3. 内容表现

- 路径：报告 → 参与度 → 页面和屏幕
- 提取维度：页面路径
- 提取指标：页面浏览量、平均参与时长、跳出率

#### 4. 事件统计

- 路径：报告 → 参与度 → 事件
- 提取事件：`article_read_complete`, `share_click`, `search_query`

### 6.2 数据导出方法

1. 在 GA4 报告中，调整时间范围
2. 点击「导出」按钮
3. 选择格式（CSV/Google 表格/Excel）
4. 保存到本地或 Google Drive

## 七、数据质量监控

### 7.1 异常告警配置

在 GA4 中设置自定义告警：

1. 进入「管理」→「自定义告警」
2. 创建告警规则：

| 告警名称 | 条件 | 阈值 |
|----------|------|------|
| 流量骤降 | 用户数环比 | < -30% |
| 流量暴涨 | 用户数环比 | > +50% |
| 跳出率异常 | 跳出率 | > 80% |
| PV 异常 | 页面浏览量 | < 100/天 |

### 7.2 告警通知设置

- 通知频率：每日一次
- 通知方式：邮件
- 接收人：运营人员邮箱
