## 二、Google Analytics 4 配置

### 2.1 创建 GA4 属性

1. 访问 [Google Analytics](https://analytics.google.com/)
2. 点击「管理」→「创建属性」
3. 选择「GA4」属性类型
4. 填写属性名称：`shenhuanjie.github.io`
5. 设置时区和货币

### 2.2 获取 Measurement ID

1. 在 GA4 属性中，点击「数据流」
2. 添加 Web 数据流
3. 填写网站 URL：`https://shenhuanjie.github.io`
4. 获取 **Measurement ID**（格式：`G-XXXXXXXXXX`）

### 2.3 Hexo 主题配置

#### 方式一：通过主题配置文件（推荐）

在 `_config.yml` 中添加配置占位符：

```yaml
# Google Analytics 4 配置
# 请替换为您的实际 Measurement ID
google_analytics:
  enable: true
  measurement_id: G-XXXXXXXXXX
  # 高级设置
  options:
    anonymize_ip: true          # 匿名化 IP（GDPR 合规）
    send_page_view: true        # 自动发送页面浏览
    debug_mode: false           # 调试模式
```

#### 方式二：通过主题主题文件注入

如果主题支持，在 `themes/chic/_config.yml` 中配置：

```yaml
google_analytics:
  enable: true
  measurement_id: G-XXXXXXXXXX
```

### 2.4 手动添加 GA4 代码

如果主题不直接支持 GA4 配置，可在 `themes/chic/layout/_partial/head.ejs` 或 `themes/chic/layout/layout.ejs` 中添加：

```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  // 配置参数
  gtag('config', 'G-XXXXXXXXXX', {
    'anonymize_ip': true,
    'send_page_view': true
  });
</script>
```

## 四、事件追踪配置

### 4.1 GA4 事件追踪

#### 阅读完成率追踪

在文章页面底部添加滚动监听脚本：

```html
<script>
  // 文章阅读完成率追踪
  let hasTrackedScroll = false;

  window.addEventListener('scroll', function() {
    if (hasTrackedScroll) return;

    const scrollHeight = document.documentElement.scrollHeight - window.innerHeight;
    const scrollPercent = (window.scrollY / scrollHeight) * 100;

    // 阅读完成（滚动超过 90%）
    if (scrollPercent > 90 && !hasTrackedScroll) {
      hasTrackedScroll = true;
      gtag('event', 'article_read_complete', {
        'event_category': 'engagement',
        'event_label': document.title,
        'value': 1
      });
    }

    // 阅读进度（每 25% 记录一次）
    const quarters = [25, 50, 75];
    quarters.forEach(q => {
      if (scrollPercent >= q && scrollPercent < q + 5) {
        gtag('event', 'article_read_progress', {
          'event_category': 'engagement',
          'event_label': document.title,
          'value': q
        });
      }
    });
  });
</script>
```

#### 分享点击率追踪

```html
<script>
  // 分享按钮点击追踪
  document.querySelectorAll('.share-button').forEach(button => {
    button.addEventListener('click', function() {
      const platform = this.dataset.platform || 'unknown';
      gtag('event', 'share_click', {
        'event_category': 'social',
        'event_label': platform,
        'value': 1
      });
    });
  });
</script>
```

### 4.2 自定义事件列表

| 事件名称 | 触发条件 | 用途 |
|----------|----------|------|
| `article_read_complete` | 阅读进度 > 90% | 文章完成率分析 |
| `article_read_progress` | 每 25% 进度 | 阅读深度分析 |
| `share_click` | 点击分享按钮 | 分享渠道分析 |
| `search_query` | 提交搜索 | 搜索关键词分析 |
| `external_link_click` | 点击外部链接 | 跳出链接分析 |

## 五、转化漏斗分析

### 5.1 定义转化漏斗

建议的漏斗模型：

```
首页访问 → 文章列表 → 文章详情 → 阅读完成 → 分享/互动
```

### 5.2 GA4 漏斗配置

在 GA4 中通过「配置」→「漏斗」创建：

1. **步骤 1：首页访问** - 选择「页面浏览」事件，路径包含 `/`
2. **步骤 2：文章列表** - 路径包含 `/archives` 或 `/categories`
3. **步骤 3：文章详情** - 路径匹配 `/:year/:month/:day/:hash/`
4. **步骤 4：阅读完成** - 触发 `article_read_complete` 事件
5. **步骤 5：分享/互动** - 触发 `share_click` 事件

## 八、常见问题

### Q1: GA4 代码添加后不生效？

1. 确认 Measurement ID 正确
2. 清除浏览器缓存
3. 使用 GA4 Debug 模式验证
4. 检查浏览器控制台是否有错误

### Q2: 如何验证追踪是否正常？

1. 安装 [GA4 DebugView](https://chrome.google.com/webstore/detail/google-analytics-debugger/jnkmfdileelhofjcijadephnlnkepbka) 扩展
2. 访问网站并触发事件
3. 在 DebugView 中查看实时事件

### Q3: 静态博客能追踪哪些指标？

| 指标 | 支持情况 |
|------|----------|
| 页面浏览量 (PV) | ✅ 完全支持 |
| 独立访客数 (UV) | ✅ 完全支持 |
| 用户停留时长 | ✅ 通过事件估算 |
| 跳出率 | ✅ 完全支持 |
| 滚动完成率 | ⚠️ 需手动配置 |
| 表单提交 | ⚠️ 需手动配置 |
