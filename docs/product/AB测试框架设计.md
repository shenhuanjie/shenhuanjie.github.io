## 一、A/B 测试框架概述

### 1.1 框架设计目标

```
┌─────────────────────────────────────────────────────────────────┐
│                      A/B 测试框架架构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │   实验曝光  │ →  │   用户分组  │ →  │   转化追踪  │         │
│  │  Exposure   │    │  Variant    │    │ Conversion  │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│                                                                 │
│  核心组件:                                                     │
│  - abTest.getVariant()     获取实验组别                         │
│  - abTest.trackExposure()  追踪实验曝光                         │
│  - abTest.trackConversion()追踪实验转化                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 框架核心功能

| 功能 | 说明 |
|------|------|
| 用户分组 | 50/50 随机分组，持久化到 localStorage |
| 实验曝光 | 记录用户进入实验的时间 |
| 转化追踪 | 追踪预设的转化事件 |
| 多实验并行 | 支持同时运行多个实验 |

## 二、框架 API

### 2.1 核心方法

```javascript
// 获取实验组别 (A 或 B)
var variant = window.__abTest.getVariant('experiment_id');

// 追踪实验曝光
window.__abTest.trackExposure('experiment_id', 'experiment_name');

// 追踪实验转化
window.__abTest.trackConversion('experiment_id', 'event_name', {
    key: 'value'
});
```

## 六、统计分析

### 6.1 关键指标

| 指标 | 计算方法 |
|------|----------|
| 转化率 | 转化用户数 / 曝光用户数 |
| 提升度 | (实验组转化率 - 对照组转化率) / 对照组转化率 |
| 统计显著性 | p < 0.05 认为显著 |

### 6.2 最小样本量计算

```
公式: n = 16 * σ² / δ²

其中:
- σ = 标准差
- δ = 期望检测的最小差异
- n = 每组所需样本量

示例 (期望检测 5% 提升，基础转化率 10%):
- p1 = 0.10 (对照组)
- p2 = 0.105 (实验组)
- n ≈ 30,000 每组
```

### 6.3 实验周期

| 场景 | 建议周期 |
|------|----------|
| 高流量 (>1000 UV/天) | 1-2 周 |
| 中流量 (300-1000 UV/天) | 2-4 周 |
| 低流量 (<300 UV/天) | 4-8 周 |

## 七、实验运行指南

### 7.1 启动新实验

1. 在 `post.ejs` 中注册实验
2. 使用 URL 参数 `?v=A` 或 `?v=B` 进行手动测试
3. 验证 localStorage 分组正确
4. 开启 GA4 DebugView 验证事件

### 7.2 监控实验

- 每日检查实验数据
- 确保样本量充足
- 监控异常波动

### 7.3 结束实验

- 达到统计显著性后停止
- 分析实验结果
- 决定是否全量上线
- 清理实验代码

## 八、注意事项

### 8.1 技术注意事项

- 用户分组需持久化，避免刷新页面重新分组
- 避免同时运行相互干扰的实验
- 实验代码需兼容 SSR/SSG 构建

### 8.2 业务注意事项

- 一个用户只能在一个实验组中
- 长期实验需考虑用户疲劳
- 敏感实验需合规审查
