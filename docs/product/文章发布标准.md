# 文章发布标准

## 一、Frontmatter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 文章标题 |
| date | 是 | 发布日期（格式：YYYY-MM-DD HH:mm:ss） |
| updated | 否 | 更新日期（格式：YYYY-MM-DD HH:mm:ss） |
| description | 是 | 文章描述（SEO 友好，80-150 字） |
| cover | 否 | 封面图片路径 |
| permalink | 否 | 自定义 URL |
| comments | 否 | 是否开启评论（默认 true） |
| toc | 否 | 是否开启目录（默认 true） |
| categories | 是 | 分类（只允许一个） |
| tags | 是 | 标签（允许多个） |
| ai_summary | 否 | AI 摘要（150 字以内，自动生成） |
| ai_generated | 否 | 是否 AI 生成 |
| source | 否 |文章来源（如：CSDN、博客园） |
| source_link | 否 | 原文链接 |
| author | 否 | 作者 |

## 二、分类标签规范

### 2.1 分类（Categories）规范

**命名规范：**
- 使用中文分类名（与系统自动分类保持一致）
- 每个文章只能属于一个分类

**标准分类列表：**

| 分类 | 适用场景 | 关键词检测 |
|-------------|---------|--------|
| AI人工智能 | 默认分类，所有 AI 文章 | AI |
| 大模型 | 大语言模型、LLM、GPT 相关 | 大模型、LLM、GPT、Claude、Gemini、文心一言、通义千问 |
| AI编程 | 编程开发、代码生成工具 | 编程、开发、code、cursor、copilot、v0、bolt |
| AI应用 | AI 应用、工具、产品使用 | 应用、app、工具、软件 |
| AI创业 | 创业、融资、投资 | 创业、融资、投资、商业 |
| AI安全 | 安全、隐私、监管 | 安全、隐私、风险、监管、伦理 |
| AI技术 | 技术、算法、模型训练 | 技术、算法、模型、训练、推理 |
| AI周报 | 周报、周总结 | 周报、周、weekly |
| AI日更 | 日更、资讯快讯 | 日更、日报、今日、最新 |

**自动分类说明：**
系统会根据文章标题和内容自动检测关键词并添加分类，但建议作者手动确认分类准确性。

### 2.2 标签（Tags）规范

**命名规范：**
- 使用中文或英文均可
- 全小写
- 具有描述性，避免过于宽泛

**推荐标签列表：**

| 标签 | 适用场景 | 关键词检测 |
|------|---------|--------|
| 大模型 | 大语言模型 | 大模型、llm、gpt、claude |
| LLM | 大语言模型 | llm、gpt、chatgpt |
| Agent | 智能体 | agent、智能体、agentic |
| RAG | 检索增强生成 | rag、检索增强 |
| 深度学习 | 深度学习 | 深度学习、deep learning、transformer |
| 机器学习 | 机器学习 | 机器学习、machine learning |
| AI编程 | 编程工具 | 编程、cursor、copilot、v0、bolt |
| AI图像 | 图像生成 | 图像、stable diffusion、midjourney、dall-e |
| AI视频 | 视频生成 | 视频、sora、runway |
| AI音频 | 语音合成 | 音频、语音、tts |
| AI安全 | 安全隐私 | 安全、隐私、对齐 |
| AI创业 | 创业融资 | 创业、融资、投资 |

## 三、图片规范

### 3.1 图片格式

| 格式 | 优先级 | 说明 |
|------|-------|------|
| WebP | 首选 | 更小体积，更高质量 |
| JPG | 次选 | 兼容性更好 |

### 3.2 图片尺寸

- **最小宽度：** 600px（确保清晰度）
- **宽高比：** 建议 16:9 或 4:3（封面图）

### 3.3 图片大小

- **最大文件大小：** 500KB
- **优化建议：** 使用 tinypng 或 squoosh 压缩

### 3.4 图片引用规范

```markdown
<!-- 推荐格式 -->
![图片描述](/images/article/2026/05/image-name.webp)

<!-- 禁止使用 -->
<!-- 外部图床链接（已废弃） -->
<!-- localhost/127.0.0.1 链接 -->
```

### 3.5 图片命名规范

- 使用有意义的名称：`article-title-feature.webp`
- 包含日期：`2026-05-09-feature.webp`
- 避免中文和特殊字符

## 四、摘要规范

### 4.1 Description 字段说明

`description` 是文章的 SEO 描述，会显示在：
- 搜索引擎结果页面（SERP）
- 社交媒体分享预览
- 文章顶部摘要

### 4.2 摘要撰写规范

| 要求 | 说明 |
|------|------|
| 长度 | 80-150 字，不超过 200 字 |
| 内容 | 概括文章核心内容，吸引点击 |
| 格式 | 纯文本，避免 HTML 标签 |
| 关键词 | 自然融入主要关键词 |

### 4.3 摘要生成策略

**优先级：AI 自动生成 > 手动撰写 > 自动截取**

1. **AI 自动生成（推荐）**
   - 使用 AI 工具根据文章内容生成
   - 确保符合长度要求
   - 包含核心关键词

2. **手动撰写**
   - 作者根据文章内容手动编写
   - 确保语言简洁、有吸引力

3. **自动截取**
   - 仅作为兜底方案
   - 取文章开头 150 字
   - 可能无法准确概括内容

### 4.4 摘要示例

**Good:**
```yaml
description: 深入解析 2026 年大模型技术发展全景，涵盖 GPT-5、Claude 4、Gemini 2 等主流模型的性能对比、应用场景及未来趋势。为开发者和研究者提供全面的技术参考。
```

**Bad:**
```yaml
# 过长
description: 本文深入分析了2026年人工智能领域的最新发展情况，包括大语言模型、图像生成模型、语音识别模型等多个领域的最新进展和技术突破，同时探讨了这些技术对各行业的影响...

# 包含 HTML 标签
description: <p>这是一篇关于 AI 的文章...</p>

# 过短
description: AI 文章
```

## 五、排版规范

### 5.1 标题层级

- 使用 `##` 作为一级标题
- 使用 `###` 作为二级标题
- 不要使用 `#` 作为文章标题（Frontmatter 的 title 已经是标题）

### 5.2 代码块

- 使用 ` ```语言 ` 指定语言
- 确保代码缩进正确

### 5.3 链接规范

- 外部链接添加 `target="_blank" rel="noopener"`
- 内部链接使用相对路径或绝对路径均可

## 六、文章质量检查清单

发布前请确认：

- [ ] Frontmatter 字段完整（title, date, description, categories, tags）
- [ ] description 长度在 80-160 字之间
- [ ] categories 只有 1 个分类
- [ ] tags 至少 1 个标签（建议3-5个）
- [ ] 图片格式为 WebP 或 JPG
- [ ] 图片宽度 >= 600px
- [ ] 图片大小 <= 500KB
- [ ] 无外部图床失效图片
- [ ] 无 localhost/127.0.0.1 链接
- [ ] 正文无 HTML 标签（如有 description 字段中的）
- [ ] 标题层级正确（##, ###）
- [ ] 代码块标注语言类型
- [ ] 前言/摘要 100-200字
- [ ] 要点列表 3-5 个
- [ ] 相关阅读 3-5 篇内部链接

## 七、标准文章结构模板（v2.15 新增）

### 7.1 文章结构规范

```yaml
# 标准文章结构

keywords: "关键词（最多5个）"

  - 摘要/前言（100-200字）
  - 要点列表（3-5个）
  - 正文（按H2/H3分层）
  - 总结（呼应要点）
```
