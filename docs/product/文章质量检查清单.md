# 文章质量检查清单

**适用范围：** 所有发布到 shenhuanjie.github.io 的文章
**最后更新：** 2026-05-08

---

## 一、标题规范

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 长度 | 中文标题 15-30 字，英文标题 5-10 个单词 | □ |
| 格式 | 不要使用特殊字符（`*`、`#`、`_` 等装饰符） | □ |
| 关键词 | 标题前 10 个字包含核心关键词 | □ |
| 大小写 | 英文单词首字母大写（专有名词除外） | □ |
| 可读性 | 避免生造词、网络用语 | □ |

**示例：**
- ✅ `2026年5月 AI资讯周报（第1周）`
- ✅ `Ollama 本地部署指南：快速搭建 AI 模型服务`
- ❌ `🔥震惊！AI圈发生大事❗`

---

## 二、摘要规范

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 字数 | 中文 80-150 字，英文 50-100 词 | □ |
| 格式 | 使用纯文本，避免 HTML 标签 | □ |
| 内容 | 包含文章核心主题、解决什么问题 | □ |
| 关键词 | 自然嵌入 2-3 个核心关键词 | □ |
| 结尾 | 不要使用"点击查看全文"等引导语句 | □ |

---

## 三、正文内容规范

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 结构 | 包含引言、正文、总结三个部分 | □ |
| 段落 | 每段不超过 4 句话，保持简洁 | □ |
| 列表 | 使用有序/无序列表组织信息 | □ |
| 强调 | 关键词加粗，不要全文大量加粗 | □ |

---

## 四、图片规范

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 格式 | 使用 WebP、PNG、JPG 格式 | □ |
| 尺寸 | 宽度不超过 1200px（ Retina 屏可 2400px） | □ |
| 大小 | 单张图片不超过 500KB | □ |
| 命名 | 使用描述性名称（如 `ollama-install-step1.png`） | □ |
| Alt 文本 | 每张图片添加 `alt` 描述（10-30 字） | □ |
| 位置 | 图片居中显示，与上下文相关 | □ |

**图片压缩建议：**
- 使用 [TinyPNG](https://tinypng.com/) 压缩 JPG/PNG
- 使用 [Squoosh](https://squoosh.app/) 压缩 WebP

```

## 七、元信息规范

### 7.1 Frontmatter 完整字段清单

```yaml
---
title: 文章标题 (必填)
date: 2026-05-08 15:30:00  # 必填，格式：YYYY-MM-DD HH:mm:ss
updated: 2026-05-08 15:30:00  # 可选，默认使用文件修改时间
description: 文章描述，80-150字  # 必填，SEO 关键
cover: /images/covers/cover-default.png  # 建议添加
permalink: /post/article-slug.html  # 建议设置
comments: true  # 建议开启
toc: true  # 建议开启
categories:  # 必填，必须使用以下有效分类之一
  - AI大模型
  - AI应用
  - 智能化标准
  - AI周报
  - AI日更
  - AI人工智能
  - 产业观察
  - AI资讯
tags:  # 必填，3-5个相关标签
  - AI大模型
  - LLM
  - AI
---
```

**注意**：禁止使用的分类如 `AI编程`、`大模型`、`DevOps`、`Docker`、`Layui`、`机器学习` 等。

### 7.3 爬取文章质量规范

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 内容完整 | 爬取后内容必须完整，不得仅包含"请访问原文链接" | □ |
| 摘要提取 | 必须提取并填写 description 字段 | □ |
| 分类映射 | 爬取脚本必须使用有效分类列表进行映射 | □ |
| 日期格式 | 爬取后必须补全时间部分（`YYYY-MM-DD HH:mm:ss`） | □ |
| 图片下载 | 图片必须下载到本地，路径使用相对路径 | □ |

**爬取脚本输出检查清单**：
- [ ] frontmatter 包含完整字段
- [ ] date 格式为 `YYYY-MM-DD HH:mm:ss`
- [ ] categories 使用有效分类
- [ ] description 不为空
- [ ] 正文内容完整（非仅链接占位）

## 十一、文章结构模板规范（v2.15 新增）

### 11.1 标准文章结构

| 章节 | 要求 | 说明 |
|------|------|------|
| 前言/摘要 | 100-200字 | 概括文章核心内容、解决什么问题 |
| 要点列表 | 3-5个 | 使用无序列表，核心观点提炼 |
| 正文 | H2/H3分层 | 按逻辑章节组织内容 |
| 总结 | 呼应要点 | 回应前言，强化核心观点 |
| 相关阅读 | 3-5篇 | 内部链接，推荐同系列文章 |

### 11.2 结构检查清单

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 前言存在 | 100-200字 | □ |
| 要点列表 | 3-5个列表项 | □ |
| 正文分层 | 正确使用H2/H3 | □ |
| 总结章节 | 与要点呼应 | □ |
| 相关阅读 | 3-5个内部链接 | □ |

### 11.3 前言撰写规范

```markdown
## 前言

[核心主题介绍，50-80字]
[解决什么问题，30-50字]
[本文将...，20-30字]
```

### 11.4 要点列表规范

```markdown
## 核心要点

- **要点1**：简洁描述（10-20字）
- **要点2**：简洁描述（10-20字）
- **要点3**：简洁描述（10-20字）
```

### 12.3 代码块规范示例

````markdown
```python
# 第1行：导入模块
from langchain import LLMChain

# 第2行：定义Prompt模板

# 第3行：创建链

# 执行：生成内容
result = chain.run(topic="人工智能")

````

## 十四、文章长度标准（v2.15 新增）

### 14.1 长度参考标准

| 关键词竞争度 | 推荐字数 | 适用场景 |
|-------------|---------|----------|
| 高竞争度 | 3000-5000字 | 热门话题、核心关键词 |
| 中竞争度 | 2000-3000字 | 细分领域、技术教程 |
| 低竞争度 | 1000-2000字 | 长尾词、新兴话题 |
| 资讯类 | 500-1000字 | 新闻快讯、周报 |

### 14.2 长度检查清单

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 正文字数 | 符合竞争度要求 | □ |
| 段落长度 | 每段不超过4句话 | □ |
| H2数量 | 不少于3个 | □ |
| 内部链接 | 不少于3个 | □ |
| 代码块 | 必要时使用，不堆砌 | □ |

### 14.3 阅读时间估算

```
阅读时间（分钟）= 正文字数 / 400（中速阅读）

推荐阅读时间：3-15分钟
- 入门教程：5-10分钟
- 深度分析：10-15分钟
- 资讯周报：3-5分钟
```
