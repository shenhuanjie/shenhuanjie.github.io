## 一、当前导航问题分析

### 1.1 现有移动端导航结构

基于 `header.ejs` 分析，当前移动端导航存在以下问题：

| 问题 | 当前实现 | 影响 |
|-----|---------|-----|
| 导航入口不明确 | 汉堡菜单 + "Menu" 文字 | 用户发现度低 |
| 层级过深 | 菜单需点击展开 | 操作步骤多 |
| 无快捷入口 | 所有导航项都在二级菜单 | 常用功能访问慢 |
| 夜间模式切换不便 | 需进入菜单才能切换 | 体验差 |
| 无分享入口 | 分享功能缺失 | 内容传播困难 |

### 1.2 用户行为数据

参考迭代规划中的目标：
- 当前移动端导航体验：**一般**
- v2.16 目标：移动端导航体验 **优秀**
- 目标：提升移动端用户留存率

## 三、交互设计规范

### 3.1 手势交互

| 交互 | 触发方式 | 响应 |
|-----|---------|-----|
| 下拉刷新 | 顶部下拉 > 50px | 加载最新文章列表 |
| 上拉加载 | 底部上拉 > 50px | 加载更多文章 |
| 双击顶部 | 快速点击顶部区域 2 次 | 滚动到页面顶部 |
| 左滑返回 | 文章详情页左滑 | 返回列表页 |
| 侧滑打开 | 从屏幕左边缘右滑 | 打开侧边栏 |


## 五、路由与状态管理

### 5.1 页面标识

```javascript
// 底部导航高亮逻辑
const currentPath = window.location.pathname;
const navItems = document.querySelectorAll('.bottom-nav .nav-item');

navItems.forEach(item => {
  const href = item.getAttribute('href');
  if (currentPath === href || currentPath.startsWith(href + '/')) {
    item.classList.add('active');
  } else {
    item.classList.remove('active');
  }
});
```

### 5.2 导航状态持久化

```javascript
// 记录用户最近访问的导航项
localStorage.setItem('lastActiveNav', 'home');

// 恢复上次访问位置
document.addEventListener('DOMContentLoaded', () => {
  const lastNav = localStorage.getItem('lastActiveNav');
  if (lastNav) {
    document.querySelector(`[data-page="${lastNav}"]`)?.classList.add('active');
  }
});
```
