## 三、frontmatter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 英文标题，使用英文标点符号 |
| date | 是 | 发布日期，建议与中文版保持同步 |
| author | 是 | 作者名（英文） |
| description | 是 | 英文描述，**需要重新撰写**，不能机翻 |
| cover | 否 | 封面图片 URL |
| tags | 是 | 英文标签 |
| categories | 是 | 英文分类（使用 ai-llm, ai-application 等） |
| lang | 是 | 必须设置为 `en` |
| translation_of | 否 | 原文路径（翻译文章必填） |
| toc | 是 | 建议开启，方便英文读者导航 |
| comments | 是 | 建议开启 |

## 六、翻译流程规范

### 6.1 翻译流程

```
1. 选题筛选
   ↓
2. 创建英文文章框架（复制 frontmatter 模板）
   ↓
3. 撰写英文标题和描述
   ↓
4. 翻译正文内容
   ↓
5. 转换代码示例（如有）
   ↓
6. 添加英文目录（Table of Contents）
   ↓
7. 关联原文（设置 translation_of）
   ↓
8. 预览检查
   ↓
9. 发布
```

### 6.2 翻译要点

1. **标题翻译**
   - 使用英文表达习惯
   - 可适度调整使其更吸引英文读者
   - 示例：「重磅发布」→「Released」

2. **描述撰写**
   - **必须重新撰写**，不能机翻
   - 简洁明了，150-160 字符
   - 包含关键词

3. **正文翻译**
   - 保持技术准确性
   - 代码示例保持原样
   - 添加英文注释说明

4. **术语处理**
   - 保持术语一致性
   - 首次出现时附中文原词
   - 使用英文技术社区常用表达

5. **图片处理**
   - 使用英文图片替代（如需要）
   - 更新图片 alt 文本为英文

### 6.3 质量检查清单

- [ ] 标题语法正确
- [ ] 描述无语法错误
- [ ] 正文流畅可读
- [ ] 代码示例完整
- [ ] 标签分类正确
- [ ] translation_of 正确设置（翻译文章）
- [ ] 链接正确

## 七、多语言切换机制

### 7.1 语言切换器

主题已内置语言切换组件 `language-switcher.ejs`，根据 page.lang 自动显示切换链接：

- 中文页面：显示「EN」链接
- 英文页面：显示「中文」链接

### 7.2 hreflang 配置

当前已配置 hreflang 标签：
```html
<link rel="alternate" hreflang="zh-CN" href="canonical_url">
<link rel="alternate" hreflang="x-default" href="canonical_url">
```

英文页面会自动映射到 `/en/` 路径，实现多语言 SEO 优化。

## 八、英文 sitemap

博客已配置 sitemap 插件多语言支持：
```yaml
sitemap:
  i18n:
    enable: true
    default_language: zh
    languages: ['zh', 'en']
```

发布英文文章后，sitemap 会自动包含 `/en/` 路径的页面。
