## 一、概述

## 三、数据分析部分（v2.2 Phase 3 新增）

### 3.1 数据分析维度

#### 流量分析

| 分析维度 | 指标 | 分析目的 |
|----------|------|----------|
| 总量分析 | PV、UV、会话数 | 了解整体规模 |
| 趋势分析 | 环比、同比 | 判断增长态势 |
| 渠道分析 | 来源分布 | 优化流量获取 |
| 内容分析 | TOP 文章 | 指导内容方向 |

#### 用户行为分析

| 分析维度 | 指标 | 分析目的 |
|----------|------|----------|
| 参与深度 | 阅读完成率、滚动进度 | 评估内容质量 |
| 互动行为 | 分享、评论、收藏 | 评估用户粘性 |
| 跳出分析 | 跳出率、退出页 | 优化页面结构 |

### 3.2 数据分析方法

#### 环比分析

```
环比变化 = (本期 - 上期) / 上期 × 100%
```

| 指标 | 本周 | 上周 | 环比 | 判断标准 |
|------|------|------|------|----------|
| PV | - | - | - | >0% 为正增长 |
| UV | - | - | - | >0% 为正增长 |
| 跳出率 | - | - | - | <0% 为改善 |

#### 完成率分析

```
阅读完成率 = article_read_complete / 文章页 PV × 100%
```

| 完成率区间 | 内容质量判断 | 优化建议 |
|------------|--------------|----------|
| < 20% | 较差 | 优化标题、提升开头吸引力 |
| 20-40% | 一般 | 增加内容深度，优化结构 |
| 40-60% | 良好 | 继续保持 |
| > 60% | 优秀 | 分析成功原因，可复制 |

#### 分享率分析

```
分享率 = share_click / 总 PV × 100%
```

| 分享率区间 | 判断 | 优化建议 |
|------------|------|----------|
| < 0.5% | 偏低 | 增加分享按钮可见性 |
| 0.5-1% | 一般 | 优化分享文案 |
| 1-3% | 良好 | 继续保持 |
| > 3% | 优秀 | 高参与度内容 |

### 3.3 异常数据识别

| 异常类型 | 判断条件 | 可能原因 |
|----------|----------|----------|
| 流量骤降 | 环比 < -30% | 算法调整、内容问题、技术故障 |
| 流量暴涨 | 环比 > +50% | 爆款内容、外部推广、爬虫 |
| 跳出率异常 | > 80% | 页面加载问题、内容不匹配 |
| 完成率骤降 | 环比 < -20% | 内容质量下降、技术问题 |

### 4.1 数据提取流程


### 4.2 GA4 数据提取步骤

#### Step 1：访问量数据

1. 登录 GA4 → 报告 → 概览
2. 设置日期范围：本周
3. 记录关键指标：用户数、页面浏览量、会话数、平均参与时长、跳出率

#### Step 2：用户获取数据

1. 报告 → 用户获取 → 流量获取
2. 导出「会话来源」数据
3. 整理各来源占比

#### Step 3：内容表现

1. 报告 → 参与度 → 页面和屏幕
2. 过滤「页面路径包含发布日期」
3. 按页面浏览量排序，导出 TOP 10
4. 查看阅读完成率数据（article_read_complete 事件）

#### Step 4：事件数据

1. 报告 → 参与度 → 事件
2. 检查自定义事件触发次数
3. 记录关键事件：`article_read_complete`, `share_click`, `external_link_click`

## 五、自动化方案（高级）

### 5.1 Google Apps Script 方案

对于技术能力较强的团队，可使用 Google Apps Script 自动抓取 GA4 数据：

```javascript
// GA4 Data API 调用示例
// 注意：需要启用 GA4 Data API 并配置 OAuth2

function getGA4Report() {
  const propertyId = 'XXXXXXXXXX'; // GA4 Property ID

  const response = UrlFetchApp.fetch(
    `https://analyticsdata.googleapis.com/v1beta/properties/${propertyId}:runReport`,
    {
      method: 'POST',
      headers: {
        Authorization: 'Bearer ' + getService().getAccessToken(),
        'Content-Type': 'application/json'
      },
      payload: JSON.stringify({
        dateRanges: [{
          startDate: '7daysAgo',
          endDate: 'today'
        }],
        metrics: [
          { name: 'screenPageViews' },
          { name: 'totalUsers' },
          { name: 'averageSessionDuration' },
          { name: 'bounceRate' },
          { name: 'engagedSessions' }
        ],
        dimensions: [{ name: 'date' }]
      })
    }
  );

  return JSON.parse(response.getContentText());
}
```

### 5.2 GitHub Actions 自动化

创建 `.github/workflows/weekly-report.yml`：

```yaml
name: Weekly Report Generation

on:
  schedule:
    # 每周一 09:00 UTC 运行 (= 17:00 北京时间)
    - cron: '0 9 * * 1'
  workflow_dispatch: # 手动触发

jobs:
  generate-report:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Fetch GA4 data
        run: node scripts/fetch-ga4-data.js
        env:
          GA4_PROPERTY_ID: ${{ secrets.GA4_PROPERTY_ID }}
          GA4_CREDENTIALS: ${{ secrets.GA4_CREDENTIALS }}

      - name: Generate weekly report
        run: node scripts/generate-weekly-report.js

      - name: Commit report
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git add docs/运营/周报/
          git commit -m "chore: 生成 $(date +%Y-%m-%d) 周报" || echo "No changes to commit"

      - name: Push changes
        uses: ad-m/github-push-action@master
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
```

### 5.3 数据抓取脚本示例

创建 `scripts/fetch-ga4-data.js`：

```javascript
/**
 * GA4 数据抓取脚本
 * 用法: node scripts/fetch-ga4-data.js
 *
 * 前置条件:
 * 1. 设置 GA4_PROPERTY_ID 环境变量
 * 2. 设置 GA4_CREDENTIALS 环境变量（OAuth2 Token 或 Service Account Key）
 */

const https = require('https');
const fs = require('fs');
const path = require('path');

// 获取环境变量
const propertyId = process.env.GA4_PROPERTY_ID;
const credentials = process.env.GA4_CREDENTIALS;

if (!propertyId || !credentials) {
  console.error('请设置 GA4_PROPERTY_ID 和 GA4_CREDENTIALS 环境变量');
  process.exit(1);
}

// API 配置
const baseUrl = 'analyticsdata.googleapis.com';
const endpoint = `/v1beta/properties/${propertyId}:runReport`;

// 获取本周数据
async function fetchWeeklyData() {
  const dates = getWeekDateRange();

  const payload = {
    dateRanges: [{
      startDate: dates.startDate,
      endDate: dates.endDate
    }],
    metrics: [
      { name: 'screenPageViews' },
      { name: 'totalUsers' },
      { name: 'averageSessionDuration' },
      { name: 'bounceRate' },
      { name: 'engagedSessions' }
    ],
    dimensions: [{ name: 'date' }]
  };

  // 发送 API 请求
  const data = await makeRequest(endpoint, payload);

  // 处理并保存数据
  const outputPath = path.join(__dirname, '../docs/运营/周报', `ga4-data-${dates.endDate}.json`);
  fs.writeFileSync(outputPath, JSON.stringify(data, null, 2));

  console.log(`数据已保存到: ${outputPath}`);
  return data;
}

// 辅助函数
function getWeekDateRange() {
  const now = new Date();
  const dayOfWeek = now.getDay();
  const startDate = new Date(now);
  startDate.setDate(now.getDate() - dayOfWeek - 6);
  const endDate = new Date(now);
  endDate.setDate(now.getDate() - dayOfWeek);

  return {
    startDate: formatDate(startDate),
    endDate: formatDate(endDate)
  };
}

function formatDate(date) {
  return date.toISOString().split('T')[0];
}

function makeRequest(endpoint, payload) {
  return new Promise((resolve, reject) => {
    const data = JSON.stringify(payload);

    const options = {
      hostname: baseUrl,
      path: endpoint,
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': data.length,
        Authorization: `Bearer ${credentials}`
      }
    };

    const req = https.request(options, res => {
      let body = '';
      res.on('data', chunk => body += chunk);
      res.on('end', () => {
        if (res.statusCode >= 200 && res.statusCode < 300) {
          resolve(JSON.parse(body));
        } else {
          reject(new Error(`API 请求失败: ${res.statusCode} ${body}`));
        }
      });
    });

    req.on('error', reject);
    req.write(data);
    req.end();
  });
}

// 执行
fetchWeeklyData().catch(console.error);
```

## 八、常见问题

### Q1: GA4 数据导出格式？

推荐使用「CSV」或「Google 表格」格式，便于后续数据处理。

### Q2: 如何获取上周对比数据？

在 GA4 中设置「比较」时间段，选择「上周同期」即可。

### Q3: 突发事件如何记录？

在周报「本周大事记」中单独标记「异常事件」，包含时间、现象、可能原因。

### Q4: 自动化脚本配置复杂吗？

GA4 Data API 需要：
1. Google Cloud 项目启用 API
2. OAuth2 或 Service Account 认证
3. 脚本开发调试

相对复杂，建议技术团队协助完成。

### Q5: 阅读完成率数据在哪里查看？

在 GA4 中：
1. 报告 → 参与度 → 事件
2. 搜索 `article_read_complete`
3. 点击查看事件详情和参数
