## 一、漏斗概述

### 1.1 业务目标
- 提升用户 engagement 和留存率
- 增加评论、分享等互动行为
- 促进回访和社交媒体关注

### 1.2 转化漏斗模型

```
新访客 → 首页/文章页
    ↓
深度阅读（滚动 > 50%）
    ↓
互动行为（评论/分享/收藏）
    ↓
回访（7日内再次访问）
    ↓
转化（Newsletter 订阅 / 社交关注）
```

## 二、漏斗步骤定义

### 步骤 1：新访客 → 首页/文章页

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 会话数 | 用户首次访问 | GA4 `session_start` |
| 新访客数 | 首次访问用户 | GA4 `new_visitor` 事件 |
| 页面浏览量 | PV | GA4 `page_view` |

**GA4 事件配置：**
```javascript
// 新访客事件 (user-interaction.js)
gtag('event', 'new_visitor', {
  event_category: 'session',
  visit_count: 1
});
```

**目标指标：**
- 新访客占比：> 60%
- 跳出率：< 50%

### 步骤 2：深度阅读（滚动 > 50%）

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 滚动深度 | 用户滚动百分比 | `scroll_depth` 事件 |
| 深度阅读率 | 滚动 > 50% 的用户比例 | scroll_depth >= 50 |

**GA4 事件配置：**
```javascript
// 滚动深度事件 (user-interaction.js)
gtag('event', 'scroll_depth', {
  event_category: 'engagement',
  depth_percent: 50, // 50, 75, 100
  page_title: pageInfo.title,
  page_url: pageInfo.url
});
```

**目标指标：**
- 深度阅读率：> 40%
- 平均滚动深度：> 60%

### 步骤 3：互动行为（评论/分享/收藏）

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 分享次数 | 分享按钮点击 | `share_article` 事件 |
| 评论数 | 评论提交 | `comment_submit` 事件 |
| 收藏数 | 收藏操作 | `favorite_article` 事件 |

**GA4 事件配置：**
```javascript
// 分享事件
gtag('event', 'share_article', {
  event_category: 'engagement',
  method: platform, // twitter, wechat, weibo, linkedin, email, other
  page_title: pageInfo.title,
  page_url: pageInfo.url
});

// 评论事件
gtag('event', 'comment_submit', {
  event_category: 'engagement',
  action: 'submit', // click, submit
  page_title: pageInfo.title,
  page_url: pageInfo.url
});

// 收藏事件
gtag('event', 'favorite_article', {
  event_category: 'engagement',
  action: 'add', // add, remove
  page_title: pageInfo.title,
  page_url: pageInfo.url
});
```

**目标指标：**
- 分享率：> 2%
- 评论率：> 1%
- 收藏率：> 3%

### 步骤 4：回访（7日内再次访问）

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 回访数 | 7日内再次访问 | `return_visitor` 事件 |
| 回访间隔 | 距上次访问天数 | return_visitor.days_since_last_visit |
| 留存率 | 回访用户 / 新访客 | 计算值 |

**GA4 事件配置：**
```javascript
// 回访事件 (user-interaction.js)
gtag('event', 'return_visitor', {
  event_category: 'session',
  days_since_last_visit: daysSinceLastVisit,
  page_title: pageInfo.title,
  page_url: pageInfo.url
});
```

**目标指标：**
- 7日留存率：> 15%
- 30日留存率：> 8%

### 步骤 5：转化（Newsletter / 社交关注）

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 转化数 | 最终转化行为 | `conversion` 事件 |
| 转化率 | 转化数 / 会话数 | 计算值 |

**GA4 事件配置：**
```javascript
// 转化事件
gtag('event', 'conversion', {
  event_category: 'conversion',
  conversion_type: 'newsletter' // 或 'social_follow', 'other'
});
```

## 三、漏斗可视化

### 3.1 GA4 漏斗报告配置

在 GA4 管理后台配置自定义漏斗：

```
漏斗名称：用户 engagement 漏斗

步骤 1：首页访问
  条件：page_location 包含 / 或 /archives
  目标：页面浏览

步骤 2：文章阅读
  条件：scroll_depth >= 50
  目标：深度阅读

步骤 3：互动行为
  条件：share_article OR comment_submit OR favorite_article
  目标：用户互动

步骤 4：回访
  条件：return_visitor (days_since_last_visit <= 7)
  目标：留存
```

### 3.2 漏斗数据解读

| 漏斗阶段 | 当前值 | 目标值 | 优化方向 |
|----------|--------|--------|----------|
| 首页访问 → 文章阅读 | 待配置 | > 50% | 优化文章摘要、推荐算法 |
| 文章阅读 → 互动行为 | 待配置 | > 10% | 优化评论区、分享按钮位置 |
| 互动行为 → 回访 | 待配置 | > 30% | 优化内容质量、邮件通知 |
| 回访 → 转化 | 待配置 | > 5% | 优化 CTA、文案 |

## 四、实施方案

### 4.1 数据采集

所有事件通过 `user-interaction.js` 自动采集：

1. **页面浏览** - GA4 自动采集
2. **滚动深度** - `ScrollTracker` 模块
3. **分享/评论/收藏** - 现有事件追踪
4. **回访标记** - `ReturnVisitorTracker` 模块

### 4.2 漏斗分析频率

- 每日：监控核心指标
- 每周：分析漏斗变化趋势
- 每月：优化漏斗策略

### 4.3 A/B 测试建议

| 测试点 | 方案 A | 方案 B |
|--------|--------|--------|
| 分享按钮位置 | 文章底部 | 浮动侧边栏 |
| 评论区位置 | 文章底部 | 侧边栏展示 |
| CTA 文案 | "关注我们" | "加入社群" |
