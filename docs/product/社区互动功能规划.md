## 一、背景与限制

### 1.1 静态博客的技术限制

Hexo 是静态博客生成器，核心限制如下：

| 限制项 | 说明 |
|--------|------|
| 无数据库 | 文章数据以 Markdown 文件形式存储，无法动态存储用户数据 |
| 无服务端 | 纯静态文件托管，无法执行服务端逻辑 |
| 无会话管理 | 无法区分不同访问者身份 |
| 无实时通信 | WebSocket 等实时技术无法使用 |

### 1.2 可行的替代方案

通过第三方服务弥补静态博客的不足：

| 功能需求 | 推荐解决方案 |
|----------|--------------|
| 评论系统 | utterances（基于 GitHub Issues） |
| 问答功能 | GitHub Discussions |
| 用户反馈 | GitHub Issues |
| 数据统计 | Google Analytics 4 |

## 二、评论系统配置

### 2.1 当前配置：Utterances

已在主题配置中启用 utterances（`themes/Chic/_config.yml`）：

```yaml
# Utterances 评论系统
utterances:
  enable: true
  repo: shenhuanjie/shenhuanjie.github.io
  issue_term: pathname
  label: comment
  theme: github-light
  dark_theme: github-dark
```

### 2.2 Utterances 优势

| 优势 | 说明 |
|------|------|
| 开源免费 | 基于 MIT 许可证 |
| GitHub 登录 | 无需额外注册账号 |
| 轻量级 | 仅 2KB JavaScript |
| 主题适配 | 支持明暗主题自动切换 |
| 通知机制 | 评论可接收 GitHub 通知 |

### 2.3 替代方案：Gitalk

如需切换到 Gitalk，可在主题配置中添加：

```yaml
# Gitalk 评论系统
gitalk:
  enable: true
  client_id: your_client_id
  client_secret: your_client_secret
  repo: shenhuanjie.github.io
  owner: shenhuanjie
  admin: shenhuanjie
  distraction_free_mode: true
```

**注意：** Gitalk 需要 GitHub OAuth Application 注册，配置更复杂。

## 三、问答模块方案

### 3.1 使用 GitHub Discussions

**方案说明：** 利用 GitHub 仓库的 Discussions 功能实现问答

**实施步骤：**

1. 在仓库设置中启用 Discussions
2. 在博客导航栏添加「问答」入口
3. 链接指向 GitHub Discussions 页面

**配置示例：**

```yaml
nav:
  # ...
  问答: https://github.com/shenhuanjie/shenhuanjie.github.io/discussions
```

### 3.2 问答分类建议

| 分类 | 用途 |
|------|------|
| Q&A | 技术问题解答 |
| 求助 | 遇到问题寻求帮助 |
| 讨论 | 话题讨论 |
| 分享 | 知识分享 |

### 4.2 内嵌投票实现

对于简单投票，可使用 GitHub Issues + 标签实现：

```
Issue 标题：[投票] 你最喜欢哪篇文章？

选项：
- 文章 A
- 文章 B
- 文章 C

用户通过评论投票
```
