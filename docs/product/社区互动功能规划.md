## 一、背景与限制

### 1.1 静态博客的技术限制

Hexo 是静态博客生成器，核心限制如下：

| 限制项 | 说明 |
|--------|------|
| 无数据库 | 文章数据以 Markdown 文件形式存储，无法动态存储用户数据 |
| 无服务端 | 纯静态文件托管，无法执行服务端逻辑 |
| 无会话管理 | 无法区分不同访问者身份 |
| 无实时通信 | WebSocket 等实时技术无法使用 |

### 1.2 可行的替代方案

通过第三方服务弥补静态博客的不足：

| 功能需求 | 推荐解决方案 |
|----------|--------------|
| 评论系统 | utterances（基于 GitHub Issues） |
| 问答功能 | GitHub Discussions |
| 用户反馈 | GitHub Issues |
| 数据统计 | Google Analytics 4 |

## 二、评论系统配置

### 2.1 当前配置：Utterances

已在主题配置中启用 utterances（`themes/Chic/_config.yml`）：

```yaml
# Utterances 评论系统
utterances:
  enable: true
  repo: shenhuanjie/shenhuanjie.github.io
  issue_term: pathname
  label: comment
  theme: github-light
  dark_theme: github-dark
```

### 2.2 Utterances 优势

| 优势 | 说明 |
|------|------|
| 开源免费 | 基于 MIT 许可证 |
| GitHub 登录 | 无需额外注册账号 |
| 轻量级 | 仅 2KB JavaScript |
| 主题适配 | 支持明暗主题自动切换 |
| 通知机制 | 评论可接收 GitHub 通知 |

### 2.3 替代方案：Gitalk

如需切换到 Gitalk，可在主题配置中添加：

```yaml
# Gitalk 评论系统
gitalk:
  enable: true
  client_id: your_client_id
  client_secret: your_client_secret
  repo: shenhuanjie.github.io
  owner: shenhuanjie
  admin: shenhuanjie
  distraction_free_mode: true
```

**注意：** Gitalk 需要 GitHub OAuth Application 注册，配置更复杂。

## 三、问答模块方案

### 3.1 使用 GitHub Discussions

**方案说明：** 利用 GitHub 仓库的 Discussions 功能实现问答

**实施步骤：**

1. 在仓库设置中启用 Discussions
2. 在博客导航栏添加「问答」入口
3. 链接指向 GitHub Discussions 页面

**配置示例：**

```yaml
nav:
  # ...
  问答: https://github.com/shenhuanjie/shenhuanjie.github.io/discussions
```

### 3.2 问答分类建议

| 分类 | 用途 |
|------|------|
| Q&A | 技术问题解答 |
| 求助 | 遇到问题寻求帮助 |
| 讨论 | 话题讨论 |
| 分享 | 知识分享 |

## 六、精精选评论机制

### 6.1 精选评论入选标准

| 标准 | 说明 | 优先级 |
|------|------|--------|
| 言之有物 | 评论内容与文章主题相关，有实质性内容 | 必须 |
| 独到见解 | 提供不同于文章的新视角或补充信息 | 高 |
| 实践价值 | 分享实际经验或踩坑经历，对读者有帮助 | 高 |
| 建设性意见 | 提出改进建议或深入探讨问题 | 中 |
| 社区贡献 | 积极回答其他读者问题，促进讨论 | 中 |

### 6.2 精选评论筛选流程

1. **日常巡查** - 站长每周检查所有评论
2. **初步筛选** - 标记符合标准的候选评论
3. **质量评估** - 综合评估评论价值
4. **精选确认** - 添加「精选」标签（在 GitHub Issue 添加 label）
5. **积分奖励** - 通过评论中@提醒通知用户获奖

### 6.3 精选评论展示

**展示位置：**

| 位置 | 说明 |
|------|------|
| 首页侧边栏 | 「本周精选」组件 |
| 关于页面 | 精选评论历史展示 |
| 微信公众号 | 每周精选汇总推送 |

**展示格式：**

```markdown
## 本周精选评论

> 「评论内容摘要...」
> —— [评论者昵称](链接) 于 [文章标题](文章链接)

> 「评论内容摘要...」
> —— [评论者昵称](链接) 于 [文章标题](文章链接)
```

### 6.4 精选评论奖励

| 奖励类型 | 说明 |
|----------|------|
| 积分奖励 | +20 积分（虚拟） |
| 称号获得 | 「精选评论者」称号 |
| 展示激励 | 首页/关于页展示 |
|  Newsletter 推送 | 优秀精选评论有机会进入 Newsletter |

### 6.5 精选评论标识

在 utterances 评论系统中，通过添加 label 实现标识：

```yaml
utterances:
  label: comment  # 普通评论
  # 精选时手动添加 "featured" label
```

## 七、用户反馈通道

### 7.1 反馈渠道

| 渠道 | 用途 | 链接/方式 |
|------|------|----------|
| GitHub Issues | Bug 报告 | /issues/new |
| GitHub Discussions | 功能建议 | /discussions |
| 评论区 | 内容纠错 | 文章底部评论 |
| 微信公众号 | 快速联系 | 公众号留言 |

### 7.2 反馈处理时效

| 类型 | 响应时间 | 处理方式 |
|------|----------|----------|
| Bug 报告 | 48h 内 | 确认并修复 |
| 功能建议 | 7 天内 | 评估并回复 |
| 内容纠错 | 24h 内 | 核实并更正 |
| 投诉建议 | 72h 内 | 评估改进 |

### 7.3 反馈激励机制

- 有效反馈奖励：+5 积分
- 优秀建议采纳：+20 积分 + 致谢
- 积极用户：邀请进入「核心用户群」（待定）
