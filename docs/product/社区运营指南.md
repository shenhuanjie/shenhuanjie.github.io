## 一、社区定位

### 1.1 核心价值

| 维度 | 说明 |
|------|------|
| 内容定位 | AI/科技资讯 + 技术实践 |
| 目标用户 | 程序员、AI 从业者、科技爱好者 |
| 核心诉求 | 获取有价值的技术资讯和实践经验 |

### 1.2 社区调性

- **专业但不晦涩** - 技术内容通俗易懂
- **开放且友善** - 鼓励讨论，尊重不同观点
- **务实求真** - 注重实践效果，避免纸上谈兵

## 二、互动功能运营

### 2.1 评论区运营

**评论审核规则：**

| 规则 | 说明 |
|------|------|
| 禁止广告 | 纯广告、营销内容直接删除 |
| 尊重原创 | 引用他人内容需注明来源 |
| 言之有物 | 无意义灌水内容定期清理 |
| 友善交流 | 人身攻击、恶意挑衅直接删除 |

**精选评论标准：**

1. 有独到见解，提供新视角
2. 对其他读者有实质帮助
3. 分享相关实践经验
4. 提出建设性问题

**精选评论展示：**

- 位置：侧边栏或关于页面
- 频率：每周更新
- 数量：每周 1-3 条

### 2.2 问答运营

**问题分类处理：**

| 类型 | 处理方式 |
|------|----------|
| 简单咨询 | 直接回复 |
| 技术问题 | 引导评论区讨论 |
| 深入探讨 | 转为专题文章 |
| 重复问题 | 引导至已有回答 |

**优质问答转化：**

- 定期整理高频问题
- 转化为「常见问题」文章
- 优秀问答内容授权后发布

## 三、用户贡献机制

### 3.1 贡献类型

| 类型 | 方式 | 奖励 |
|------|------|------|
| 内容贡献 | 投稿、纠错 | 积分奖励 |
| 互动贡献 | 优质评论 | 精选展示 |
| 传播贡献 | 社交分享 | 积分奖励 |
| 社区贡献 | 回答问题 | 积分奖励 |

### 3.2 投稿流程

1. 通过 GitHub Issues 或评论投稿
2. 站长审核内容质量
3. 采纳后注明作者来源
4. 给予积分奖励

### 3.3 贡献榜单

**更新频率：** 每月一次

**榜单内容：**

```markdown

1. @用户名 - N条评论
2. @用户名 - N条评论
3. @用户名 - N条评论

### 分享贡献TOP3
1. @用户名 - N次分享
2. @用户名 - N次分享
3. @用户名 - N次分享

### 最佳评论
> 评论摘要 - [文章标题]
```

## 五、社区规范

### 5.1 发帖/评论规范

**鼓励：**

- 分享真实经验
- 提出建设性意见
- 帮助解答他人问题
- 传播有价值的内容

**禁止：**

- 纯广告/营销内容
- 虚假信息/谣言
- 人身攻击/恶意抹黑
- 无关外链（确需分享请说明）
- 重复灌水

### 5.2 违规处理

| 级别 | 行为 | 处理方式 |
|------|------|----------|
| 轻微 | 无关内容 | 删除+警告 |
| 中度 | 重复灌水 | 删除+禁止评论1周 |
| 重度 | 广告/人身攻击 | 删除+禁止评论1月 |
| 严重 | 违法内容 | 全部清除+拉黑 |

### 5.3 申诉机制

如对处理有异议，可通过以下方式申诉：
- 微信公众号留言
- GitHub Issue 反馈
