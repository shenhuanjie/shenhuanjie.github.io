# 用户激励体系

**创建日期：** 2026-05-08
**所属版本：** v2.1 Phase 4
**博客名称：** 程序猿视界

---

## 一、设计背景

### 1.1 静态博客的激励限制

由于 Hexo 是静态博客，无法实现：
- 真实的用户账号系统
- 实时积分计算
- 在线勋章领取

### 1.2 激励目标

| 目标 | 说明 |
|------|------|
| 提升互动率 | 鼓励评论、分享 |
| 增加停留时间 | 通过内容设计延长阅读 |
| 促进传播 | 激励读者分享文章 |

---

## 二、阅读勋章体系

### 2.1 勋章设计（虚拟展示）

由于无法自动发放，采用「成就展示页」方案

| 勋章 | 名称 | 获取条件 | 状态 |
|------|------|----------|------|
| 访客勋章 | 初次访问 | 访问网站 | 自动 |
| 读者勋章 | 阅读文章 | 阅读任意文章 | 自动 |
| 活跃读者 | 多次访问 | 访问 10+ 次 | 自动 |
| 深度读者 | 长时间阅读 | 停留 5min+ | 自动 |
| 忠实粉丝 | 持续关注 | 访问 30+ 次 | 自动 |

### 2.2 勋章展示页面

创建 `/source/trophy/index.md`：

```markdown
---
title: 成就中心
layout: trophy
---

# 成就中心

感谢你的支持！

## 已解锁成就

- [x] 初次访问
- [x] 阅读文章

## 成就徽章

待实现功能：基于访问数据分析自动解锁
```

### 2.3 后续实现方案

当集成 Google Analytics 4 后，可通过以下方式实现自动化：

```javascript
// GA4 事件追踪示例
gtag('event', 'achievement_unlock', {
  achievement_id: 'first_visit',
  achievement_name: '初次访问'
});
```

## 四、社区积分系统

### 4.1 积分体系（虚拟）

由于静态博客限制，采用「虚拟积分」设计

| 行为 | 积分 | 说明 |
|------|------|------|
| 访问网站 | +1 | 每日首次访问 |
| 阅读文章 | +2 | 完成阅读 |
| 评论 | +5 | 每条评论 |
| 被精选评论 | +20 | 被站长精选 |
| 分享文章 | +10 | 每篇分享 |
| 关注公众号 | +15 | 关注后截图反馈 |

### 4.2 积分展示

在侧边栏或关于页面展示（待实现）：

```markdown
## 我的积分

- 当前积分：待实现
- 排名：待实现

[查看积分规则](/docs/积分规则.md)
```

### 4.3 积分用途

| 用途 | 所需积分 |
|------|----------|
| 精选评论置顶 | 100 |
| 专属称号 | 200 |
| 线下活动邀请 | 500 |

## 五、用户贡献榜单

### 5.1 榜单设计

**展示位置：** 关于页面或专属页面

**榜单类型：**

| 榜单 | 维度 | 更新频率 |
|------|------|----------|
| 评论贡献榜 | 评论数量 | 每月 |
| 分享贡献榜 | 分享次数 | 每月 |
| 活跃访客榜 | 访问频率 | 每月 |

### 5.2 精选评论展示

**规则：**

- 每周精选 1-3 条优质评论
- 入选条件：言之有物、有帮助
- 展示位置：首页侧边栏或关于页面

**展示格式：**

```markdown
## 本周精选评论

> [文章标题] - 评论摘要...
> —— 评论者昵称

> [文章标题] - 评论摘要...
> —— 评论者昵称
```
