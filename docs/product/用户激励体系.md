# 用户激励体系

**创建日期：** 2026-05-08
**所属版本：** v2.1 Phase 4
**博客名称：** 程序猿视界

---

## 一、设计背景

### 1.1 静态博客的激励限制

由于 Hexo 是静态博客，无法实现：
- 真实的用户账号系统
- 实时积分计算
- 在线勋章领取

### 1.2 激励目标

| 目标 | 说明 |
|------|------|
| 提升互动率 | 鼓励评论、分享 |
| 增加停留时间 | 通过内容设计延长阅读 |
| 促进传播 | 激励读者分享文章 |

## 二、阅读勋章体系

### 2.1 勋章设计（虚拟展示）

由于无法自动发放，采用「成就展示页」方案

| 勋章 | 名称 | 获取条件 | 状态 |
|------|------|----------|------|
| 访客勋章 | 初次访问 | 访问网站 | 自动 |
| 读者勋章 | 阅读文章 | 阅读任意文章 | 自动 |
| 活跃读者 | 多次访问 | 访问 10+ 次 | 自动 |
| 深度读者 | 长时间阅读 | 停留 5min+ | 自动 |
| 忠实粉丝 | 持续关注 | 访问 30+ 次 | 自动 |

### 2.2 勋章展示页面

创建 `/source/trophy/index.md`：

```markdown
---
title: 成就中心
layout: trophy
---

# 成就中心

感谢你的支持！

## 已解锁成就

- [x] 初次访问
- [x] 阅读文章

## 成就徽章

待实现功能：基于访问数据分析自动解锁
```

### 2.3 后续实现方案

当集成 Google Analytics 4 后，可通过以下方式实现自动化：

```javascript
// GA4 事件追踪示例
gtag('event', 'achievement_unlock', {
  achievement_id: 'first_visit',
  achievement_name: '初次访问'

```

### 6.2 勋章获取机制

**自动获取（已实现）：**

- 访客勋章：基于 localStorage 首次访问标记
- 读者勋章：基于首次阅读文章事件
- 评论达人：基于 GitHub 账号评论统计（需手动核实）

**待实现功能：**

- 深度读者：需要集成 GA4 停留时间追踪
- 忠实粉丝：需要长期访问频率追踪
- 分享达人：需要分享按钮点击事件追踪
