## 一、当前问题分析

### 1.1 移动端性能现状

基于对现有主题代码的分析，发现以下性能瓶颈：

| 瓶颈类型 | 当前问题 | 影响程度 |
|---------|---------|---------|
| 图片资源 | 文章内图片未做响应式处理，移动端加载原图 | 高 |
| CSS/JS | 所有样式和脚本全量加载，无代码分割 | 中 |
| 字体加载 | 字体文件较大，首次加载延迟 | 中 |
| CDN | 已启用 jsDelivr，但资源映射不完整 | 低 |
| 渲染阻塞 | JavaScript 放置在 body 末尾，但 CSS 在 head | 中 |

### 1.2 性能指标目标

| 指标 | 当前值 | 目标值 | 优化策略 |
|-----|-------|-------|---------|
| First Contentful Paint (FCP) | ~2.5s | <1.5s | 关键CSS内联 |
| Largest Contentful Paint (LCP) | ~3.5s | <2.0s | 图片优化+CDN |
| Time to Interactive (TTI) | ~3.0s | <2.0s | 代码分割 |
| Cumulative Layout Shift (CLS) | ~0.15 | <0.1 | 图片尺寸预设 |

## 二、图片压缩方案

### 2.1 响应式图片策略

```html
<!-- 使用 srcset 实现响应式图片 -->
<img
  src="image-800w.jpg"
  srcset="
    image-400w.jpg 400w,
    image-800w.jpg 800w,
    image-1200w.jpg 1200w
  "
  sizes="(max-width: 480px) 400px,
         (max-width: 768px) 800px,
         1200px"
  alt="描述文字"
  loading="lazy"
  width="800"
  height="600"
/>
```

### 2.2 图片格式优化

| 场景 | 推荐格式 | 压缩工具 |
|-----|---------|---------|
| 照片/截图 | WebP (>90%压缩率) | cwebp, squoosh |
| 图标/UI元素 | SVG (无限缩放) | SVGO |
| 动图 | WebP 动图 / AVIF | ffmpeg |
| 降级方案 | JPEG/PNG | tinypng |

### 2.3 图片加载策略

```javascript
// 懒加载实现 (基于 Intersection Observer)
const lazyImages = document.querySelectorAll('img[loading="lazy"]');
const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src || img.src;
      img.classList.add('loaded');
      observer.unobserve(img);
    }
  });
});
lazyImages.forEach(img => imageObserver.observe(img));
```

### 2.4 文章图片处理流程

```yaml
# Hexo 图片处理配置
image_processing:
  # 响应式图片生成
  responsive_images:
    widths: [400, 800, 1200]
    format: webp
    quality: 85
  # 缩略图生成
  thumbnails:
    width: 300
    height: 200
    crop: true
```

    - /css/style.css

    - /js/script.js

## 四、代码分割方案

### 4.1 CSS 代码分割

```stylus
// base.styl - 始终加载
// layout.styl - 始终加载
// font.styl - 始终加载
// variable.styl - 始终加载

// mobile.styl - 仅移动端加载
@media screen and (max-width: 768px)
  @import 'mobile-optimizations'

// dark-theme.styl - 按需加载
.dark-theme &
  @import 'dark-theme-extensions'
```

### 4.2 JavaScript 代码分割

```javascript
// script.js 模块化改造

// 主包 - 核心功能
const core = {
  init: () => { /* ... */ },
  theme: () => { /* ... */ }
};

// 懒加载模块
const modules = {
  // 评论区 - 仅文章页加载
  comments: () => import('./comments.js'),
  // 分享功能 - 按需加载
  share: () => import('./share.js'),
  // 搜索 - 仅搜索页加载
  search: () => import('./search.js')
};

// 代码高亮 - 仅需要时加载
const highlight = () => import('./highlight.js');
```

### 4.3 关键渲染路径优化

```html
<!-- 关键 CSS 内联 -->
<style>
/* 仅包含首屏渲染所需的核心样式 */
body { font-family: system-ui, sans-serif; }
.header { padding: 1rem; }
.main { max-width: 800px; margin: 0 auto; }
</style>

<!-- 非关键 CSS 异步加载 -->
<link rel="preload" href="/css/non-critical.css" as="style"
      onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="/css/non-critical.css"></noscript>
```

### 4.4 资源预加载策略

```html
<!-- 预连接关键域名 -->
<link rel="preconnect" href="https://cdn.jsdelivr.net">
<link rel="preconnect" href="https://fonts.googleapis.com">

<!-- 预加载关键资源 -->
<link rel="preload" href="/fonts/iconfont.woff2" as="font" crossorigin>
<link rel="preload" href="/css/style.css" as="style">

<!-- 预加载关键图片 -->
<link rel="preload" href="/images/hero.webp" as="image">
```

## 五、缓存策略

### 5.1 缓存配置

```yaml
# _config.yml 缓存配置
cache:
  enable: true
  # HTML 缓存 (Hexo 内置)
  html_cache: true
  # 资源缓存 (Service Worker)
  sw_cache:
    enable: true
    cache_name: 'blog-v1'
    static_prefix: ['/css/', '/js/', '/fonts/', '/images/']
    max_age: 604800  # 7天
```

### 5.2 Service Worker 缓存策略

```javascript
// sw.js 缓存策略
const CACHE_NAME = 'blog-v1';
const STATIC_ASSETS = [
  '/',
  '/css/style.css',
  '/js/script.js',
  '/fonts/iconfont.woff2'
];

// 缓存策略
const cacheStrategy = {
  // 静态资源: Cache First
  static: {
    pattern: /\.(css|js|woff2?|png|jpg|svg)$/,
    strategy: 'cache-first'
  },
  // 页面: Network First
  pages: {
    pattern: /\//,
    strategy: 'network-first',
    maxAge: 3600
  },
  // API: Network Only
  api: {
    pattern: /\/api\//,
    strategy: 'network-only'
  }
};
```
