## 一、背景与目标

### 1.1 建设背景

程序猿视界一直以站长独自创作内容为主，为丰富内容来源、提升读者参与感，需要建立完善的内容共创机制，让读者从「内容消费者」转变为「内容共创者」。

### 1.2 核心价值

| 价值维度 | 说明 |
|----------|------|
| 内容丰富 | 引入多元视角和专业深度 |
| 用户粘性 | 参与者对平台归属感更强 |
| 传播裂变 | 作者自发分享传播 |
| 品牌提升 | 专业化运营提升影响力 |

### 1.3 目标设定

| 阶段 | 时间 | 目标 |
|------|------|------|
| 初期 | 3个月 | 收到投稿 10+ 篇 |
| 成长期 | 6个月 | 采用投稿 20+ 篇 |
| 稳定期 | 12个月 | 形成稳定投稿生态 |

## 二、投稿类型定义

### 2.1 内容类型分类

| 类型 | 定义 | 字数要求 | 原创要求 |
|------|------|----------|----------|
| 技术教程 | 实战经验、技术讲解 | 2000+ | 原创 |
| 工具评测 | 工具使用、对比评测 | 1500+ | 原创 |
| 行业观察 | 趋势分析、观点输出 | 1500+ | 原创/授权转载 |
| 踩坑分享 | 问题解决、经验教训 | 1000+ | 原创 |
| 资讯投递 | 行业新闻、产品发布 | 500+ | 原创/编译 |
| 翻译文章 | 海外优质内容翻译 | 1500+ | 需授权 |

### 2.2 内容质量标准

**必须满足：**
- 技术内容准确无误
- 代码示例可正常运行
- 逻辑清晰、结构完整
- 无抄袭、侵权问题

**建议满足：**
- 有独特见解或实践价值
- 配图清晰、说明充分
- 提供延伸阅读资源

## 三、投稿流程

### 3.1 完整投稿流程

```
┌─────────────────────────────────────────────────────────┐
│                     投稿流程图                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  投稿者 ──→ 初审 ──→ 修改 ──→ 复审 ──→ 排版 ──→ 发布  │
│              │         │        │        │        │    │
│              ↓         ↓        ↓        ↓        ↓    │
│           格式检查   完善内容  技术核实  适配风格  正式发布 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 各阶段说明

**阶段一：投稿提交**
- 投稿者通过指定渠道提交
- 填写投稿登记表
- 提交完整稿件

**阶段二：初审（1-3天）**
- 检查格式是否符合要求
- 评估内容方向是否匹配
- 通知作者初审结果

**阶段三：内容修改（1-7天）**
- 提出修改意见
- 作者修改完善
- 确认最终版本

**阶段四：复审（1-2天）**
- 技术准确性核实
- 版权问题排查
- 最终质量把控

**阶段五：排版适配**
- 按博客风格排版
- 添加配图和代码高亮
- SEO基础优化

**阶段六：正式发布**
- 与作者确认发布时间
- 发布文章
- 通知作者

## 四、投稿渠道

### 4.1 投稿方式

| 渠道 | 适用类型 | 处理时效 | 说明 |
|------|----------|----------|------|
| GitHub PR | 技术长文 | 7天 | 推荐，流程规范 |
| GitHub Issue | 简短投稿/纠错 | 48h | 适合快速反馈 |
| 微信公众号 | 紧急联系 | 72h | 适合移动端投稿 |
| 邮箱 | 正式投稿 | 48h | 备用渠道 |

### 4.2 投稿模板

**模板一：GitHub PR 投稿**

```markdown
## 投稿登记表

### 作者信息
- 投稿人昵称：
- 联系方式（微信/邮箱）：
- 个人主页（可选）：

### 稿件信息
- 文章标题：
- 文章类型：[技术教程/工具评测/行业观察/踩坑分享/其他]
- 摘要（200字以内）：
- 原创声明：[本人原创/已获授权转载]

### 稿件正文
（在此粘贴完整文章内容，使用 Markdown 格式）
```

**模板二：快速投稿（Issue）**

```markdown

（如果有的话）

（方便后续沟通）
```

## 五、审核标准

### 5.1 初审标准

| 维度 | 要求 | 否决条件 |
|------|------|----------|
| 格式规范 | 符合投稿模板 | 格式严重不符 |
| 方向匹配 | 属于 AI/技术/科技范畴 | 明显偏离定位 |
| 原创性 | 原创或已授权 | 抄袭/侵权 |
| 完整性 | 内容完整、有实质价值 | 过于简陋 |

### 5.2 复审标准

| 维度 | 要求 | 否决条件 |
|------|------|----------|
| 技术准确性 | 概念正确、代码可运行 | 存在重大错误 |
| 实用价值 | 对读者有实际帮助 | 无实质内容 |
| 逻辑清晰 | 结构合理、表达清楚 | 逻辑混乱 |
| 侵权排查 | 无版权问题 | 存在侵权 |

### 5.3 审核反馈模板

**初审通过：**
```
✅ 初审通过！

您的稿件《XXX》符合我们的投稿要求，现进入修改阶段。

修改建议：
1. [建议1]
2. [建议2]
...

请在XX日前完成修改并提交。
```

**初审不通过：**
```
❌ 初审未通过

感谢您的投稿！很遗憾，您的稿件《XXX》未通过初审。

原因：[具体原因]

建议：
1. [改进建议]
2. 如有修改可重新投稿

期待您的下次投稿！
```

## 七、投稿规范

### 7.1 内容规范

**必须遵守：**
- 原创内容需为本人创作
- 转载内容需注明来源并获得授权
- 翻译内容需注明原文出处
- 不涉及政治敏感话题
- 不包含广告、营销内容

**格式要求：**
- 使用标准 Markdown 格式
- 代码块注明语言类型
- 图片使用相对路径或外链
- 段落结构清晰、层次分明

### 7.2 禁止投稿内容

- 涉及人身攻击、恶意抹黑
- 虚假信息、未经证实的谣言
- 色情低俗、暴力血腥内容
- 侵犯他人知识产权
- 明显的商业广告软文

---

## 八、质量保障

### 8.1 编辑规范

**标题规范：**
- 简洁明了，控制在25字以内
- 包含关键词，有吸引力
- 避免标题党、夸张用语

**内容规范：**
- 开头引入背景，吸引读者
- 主体部分逻辑清晰
- 结尾总结或引导互动
- 配图与内容相关、清晰美观

**SEO规范：**
- 正文中自然融入关键词
- 图片添加 Alt 文字描述
- 段落标题使用层级结构

### 8.2 常见问题处理

| 问题类型 | 处理方式 |
|----------|----------|
| 内容需大改 | 提供详细修改建议 |
| 部分内容有问题 | 指出具体问题 |
| 存在版权风险 | 要求替换或删除 |
| 质量问题严重 | 建议重新选题 |

## 九、作者服务

### 9.1 作者权益

| 权益 | 说明 |
|------|------|
| 署名权 | 保留作者昵称和个人链接 |
| 修改权 | 发布前确认最终版本 |
| 撤回权 | 发布前可随时撤回 |
| 维权权 | 遭遇侵权时协助维权 |

### 9.2 常见问题 FAQ

**Q：投稿后多久能得到回复？**
A：初审一般在1-3个工作日内，复审1-2个工作日。

**Q：投稿被拒后可以重新投吗？**
A：可以，根据反馈修改后重新提交。

**Q：是否可以一稿多投？**
A：不支持，稿件采用前请勿投往其他平台。

**Q：文章发布后会删除吗？**
A：无特殊情况不会删除，如需下架请提前联系。
