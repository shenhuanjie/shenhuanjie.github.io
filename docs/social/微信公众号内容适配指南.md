**文档版本：** v1.0
**创建日期：** 2026-05-11
**适用场景：** 将博客文章适配发布至微信公众号

## 一、封面图规范

### 1.1 尺寸要求

| 参数 | 数值 |
|------|------|
| 比例 | 2.35:1（横向封面） |
| 宽度 | 900px |
| 高度 | 383px |
| 格式 | JPG/PNG，建议 WebP |
| 文件大小 | ≤ 500KB |

### 1.2 设计要点

- **文字占比**：封面图上如有文字，文字区域应集中在中央 60% 范围
- **留白边距**：上下左右各留 10% 留白，避免被裁剪
- **配色建议**：使用高对比度配色，确保在手机端清晰可见
- **品牌元素**：右下角添加博客 Logo 水印（透明度 20-30%）

### 1.3 生成脚本

```javascript
// scripts/generate-wechat-cover.js
// 自动生成符合微信规范的封面图
const sharp = require('sharp');

async function generateWechatCover(inputPath, outputPath, options = {}) {
  const targetWidth = 900;
  const targetHeight = 383;

  await sharp(inputPath)
    .resize(targetWidth, targetHeight, {
      fit: 'cover',
      position: 'center'
    })
    .composite([{
      input: options.watermark,
      gravity: 'southeast'
    }])
    .toFile(outputPath);

  console.log(`封面图已生成: ${outputPath}`);
}
```

## 二、标题优化策略

### 2.1 标题原则

| 原则 | 说明 |
|------|------|
| 字数控制 | 14-20 字为最佳，过长会被截断 |
| 吸引点击 | 使用数字、疑问、对比等技巧 |
| 关键词前置 | 核心关键词放在前 10 个字 |
| 避免标题党 | 内容需与标题匹配，杜绝虚假夸大 |

### 2.2 标题模板

```
# 模板示例

数字型: 《[数字]个[主题]技巧，让你的[效果]》
疑问型: 《[主题]怎么做？[方法论]详解》
对比型: 《[旧方法] vs [新方法]，你选对了吗？》
热点型: 《[热点事件]背后，[核心洞察]》
```

### 2.3 标题生成规则

```javascript
// 标题生成策略
const generateWechatTitle = (articleTitle) => {
  // 1. 提取核心关键词
  const keywords = extractKeywords(articleTitle);

  // 2. 应用模板
  const templates = [
    `【深度】${articleTitle}`,
    `${keywords[0]}: ${articleTitle}`,
    `一文搞懂${articleTitle.replace(/^《|》$/g, '')}`
  ];

  // 3. 截取合适长度
  return templates[0].slice(0, 20);
};
```

## 三、导语文案规范

### 3.1 导语结构

```
[开场共鸣/痛点] → [核心价值] → [为什么值得看] → [引导阅读]
```

### 3.2 导语模板

```markdown
# 模板一：痛点引导型
你是否遇到过[痛点描述]？
本文将为你详细介绍[解决方案]，帮助[达成效果]。
建议先收藏再看，避免下次找不到。

# 模板二：价值承诺型
关于[主题]，网上90%的内容都是错的。
作为[领域]从业者，本文将带你[核心洞察]，
看完你将彻底理解[预期收获]。

# 模板三：故事型
上周，一位[用户画像]找到我，问我[问题]。
这个看似[表面现象]的问题，背后其实[深层原因]。
今天，我就来详细聊聊[主题]。
```

### 3.3 导语字数

- 字数控制在 **80-120 字**
- 段前无需缩进
- 使用表情符号增加亲和力（非必须，每段不超过 2 个）

## 四、正文格式适配

### 4.1 排版规范

| 项目 | 规范 |
|------|------|
| 正文字体 | 16px，字体 Microsoft YaHei / PingFang SC |
| 行间距 | 1.75 或 2.0 倍 |
| 段间距 | 段前 0，段后 16px |
| 首行缩进 | 无 |
| 两端缩进 | 16px |
| 图片宽度 | 最大宽度 100%，居中显示 |
| 图片说明 | 图片下方添加灰色说明文字，12px |

### 4.2 内容调整

```javascript
// 自动转换博客正文为微信格式
function adaptForWechat(htmlContent) {
  return htmlContent
    // 移除博客特有样式
    .replace(/class="blog-[^"]*"/g, '')
    // 转换图片尺寸
    .replace(/<img/g, '<img style="width:100%;display:block;margin:16px 0;"')
    // 添加图片说明占位
    .replace(/(<img[^>]*>)/g, '$1<p style="color:#999;font-size:12px;text-align:center;">插图</p>')
    // 处理代码块（微信不支持，建议截图或使用代码片段）
    .replace(/<pre>/g, '<div class="code-block">')
    .replace(/<\/pre>/g, '</div>');
}
```

### 4.3 代码处理建议

微信公众号对代码块支持较差，建议：

1. **短代码**（< 3 行）：使用 `代码` 样式
2. **中等代码**（3-10 行）：截图或使用「代码家」小程序
3. **长代码**：建议只展示核心片段，引导至原文查看

## 五、引流策略

### 5.1 文末引流模板

```markdown
---

## 📚 相关文章推荐

- [相关文章1标题](原文链接)
- [相关文章2标题](原文链接)

---

**关注「程序猿视界」**

回复以下关键词，获取更多内容：
- 回复 **AI** → 获取 AI 专题精选文章
- 回复 **效率** → 获取效率工具合集
- 回复 **加群** → 加入技术交流群

点击下方卡片关注公众号：
[图片: 二维码]
```

### 5.2 原文链接设置

在微信公众号后台「原文链接」处填写博客原文地址，便于读者跳转查看完整内容和代码。

### 5.3 转载声明

对于非首发内容，在文末添加：

```markdown
> 本文首发于[博客名称]，如需转载，请联系作者获取授权。
> 原创不易，抄袭必究。
```

## 六、发布检查清单

发布前请逐项确认：

- [ ] 封面图尺寸为 900x383（2.35:1）
- [ ] 封面图文件大小 < 500KB
- [ ] 标题字数在 14-20 字之间
- [ ] 导语已撰写（80-120 字）
- [ ] 正文格式符合排版规范
- [ ] 代码块已处理（截图或简化）
- [ ] 文末已添加引流内容
- [ ] 原文链接已填写
- [ ] 文章封面图已上传至微信后台
- [ ] 已开启「允许留言」和「允许分享」

## 八、平台特性备注

| 特性 | 说明 |
|------|------|
| 编辑器限制 | 微信编辑器功能有限，建议先在 MDnice 排版后复制 |
| 图片大小 | 单张图片不超过 10MB，建议压缩至 500KB 以内 |
| 视频限制 | 视频建议上传至腾讯视频后嵌入 |
| 链接限制 | 文中链接仅能跳转至微信生态内链接 |
| 定时发布 | 可设置定时发布，建议选择 20:00-22:00 |
