# 数据备份指南

本文档定义博客数据的备份策略、流程和恢复方案。

## 备份目标

确保博客内容在意外情况下能够快速恢复，最大限度减少数据丢失风险。

---

## 一、备份范围

### 1.1 核心数据

| 数据类型 | 路径 | 备份频率 | 保留策略 |
|----------|------|----------|----------|
| 文章源文件 | `source/_posts/` | 实时（Git） | 永久 |
| 草稿文件 | `source/_drafts/` | 实时（Git） | 永久 |
| 页面文件 | `source/` | 实时（Git） | 永久 |
| 配置文件 | `_config.yml` | 实时（Git） | 永久 |
| 主题配置 | `themes/Chic/_config.yml` | 实时（Git） | 永久 |

### 1.2 资源文件

| 数据类型 | 路径 | 备份频率 | 保留策略 |
|----------|------|----------|----------|
| 文章图片 | `source/images/posts/` | 每日 | 30天 |
| 封面图片 | `source/images/covers/` | 每周 | 90天 |
| 赞助商图片 | `source/images/sponsors/` | 每周 | 90天 |

### 1.3 生成文件

| 数据类型 | 路径 | 说明 |
|----------|------|------|
| 公共文件 | `public/` | Hexo 生成，不备份 |
| 节点模块 | `node_modules/` | npm 安装，不备份 |

---

## 二、备份策略

### 2.1 Git 备份（核心）

核心内容通过 Git 版本控制管理，托管在 GitHub。

```bash
# 手动备份流程
git add source/_posts/ source/_drafts/ source/favorites/ _config.yml themes/Chic/_config.yml
git commit -m "backup: $(date +%Y-%m-%d)"
git push origin main
```

**优势**：
- 完整的版本历史
- 异地存储
- 协作支持

### 2.2 图片备份

图片资源较大，通过定期同步到云存储。

```bash
# 图片备份脚本示例
rsync -avz --delete source/images/posts/ backup-server:/blog/images/posts/
```

### 2.3 数据库备份

本博客使用纯静态文件，无需数据库备份。

## 三、备份频率

| 备份类型 | 频率 | 触发方式 |
|----------|------|----------|
| 核心数据 | 实时 | Git commit 后自动 |
| 图片资源 | 每日 | cron job |
| 全量备份 | 每周 | cron job |

---

## 四、恢复流程

### 4.1 从 Git 恢复

```bash
# 克隆仓库
git clone https://github.com/shenhuanjie/shenhuanjie.github.io.git

# 切换到指定版本
git checkout <commit-hash>

# 安装依赖
npm install

# 生成静态文件
npm run build
```

### 4.2 图片恢复

```bash
# 从备份服务器同步
rsync -avz backup-server:/blog/images/posts/ source/images/posts/
```

## 六、注意事项

1. **敏感信息**：避免提交 `.env`、API 密钥等敏感文件
2. **大文件**：图片超过 5MB 建议使用 Git LFS
3. **频率**：重要文章发布后立即提交
4. **验证**：定期测试恢复流程
