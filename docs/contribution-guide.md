## 二 投稿方式

### 2.1 GitHub PR 投稿（推荐）

**优点**：方便追踪稿件状态，支持协作编辑

**步骤**：

1. **Fork 仓库**
   ```bash
   git clone https://github.com/shenhuanjie/shenhuanjie.github.io.git
   cd shenhuanjie.github.io
   git fork
   ```

2. **创建文章文件**
   ```bash
   # 文章放在 source/_posts/ 目录
   # 文件命名格式: YYYY-MM-DD-文章标题.md
   touch source/_posts/2026-05-11-文章标题.md
   ```

3. **编写文章 Front-matter**
   ```yaml
   ---
   title: 文章标题
   date: 2026-05-11
   categories: [AI大模型]
   tags: [AI, 技术分析]
   description: 文章摘要（150字以内）
   author: 你的昵称
   ---
   ```

4. **提交 PR**
   - Push 到你的 Fork 仓库
   - 创建 Pull Request 到主仓库
   - PR 标题格式：`[投稿] 文章标题`

### 2.2 邮件投稿

**投稿邮箱**：`shenhuanjie@email.com`

**邮件格式**：
- 主题：`【投稿】文章标题`
- 正文：
  ```
  标题：
  分类：
  标签：
  摘要：

  正文内容（支持 Markdown）
  ```

## 三 质量要求

### 3.1 内容质量

- **原创性**：必须是原创内容，拒绝抄袭和洗稿
- **技术准确性**：代码和概念需准确无误
- **实用性**：对读者有实际帮助
- **条理性**：结构清晰，逻辑通顺

### 3.2 格式规范

```markdown
---
title: 文章标题
date: YYYY-MM-DD
categories: [分类]
tags: [标签1, 标签2]
description: 摘要
author: 作者名
---

# 文章标题（h1 仅标题使用）

## 二级标题

正文内容...

## 代码示例

```javascript
// 代码块需要标注语言
const hello = 'world';
```

## 图片

- 图片使用外链或上传到 `/source/images/` 目录
- 单张图片大小不超过 2MB
```

### 3.3 禁止内容

- 政治敏感话题
- 色情低俗内容
- 暴力恐怖内容
- 广告推广信息
- 违法信息

## 四 审核流程

### 4.1 审核周期

| 投稿方式 | 审核周期 | 结果通知 |
|----------|----------|----------|
| GitHub PR | 3-5 个工作日 | PR 评论回复 |
| 邮件投稿 | 5-7 个工作日 | 邮件回复 |

### 4.2 审核标准

1. **格式检查**：Front-matter 是否完整
2. **内容审核**：原创性、技术准确性
3. **合规检查**：是否符合发布规范
4. **排版审核**：代码、图片是否规范

### 4.3 审核结果

| 结果 | 说明 | 后续操作 |
|------|------|----------|
| 通过 | 审核通过，将于 48 小时内发布 | 无需操作 |
| 修改 | 需要作者修改后重新提交 | 根据意见修改 |
| 拒绝 | 不符合发布标准 | 可申诉或另投 |

## 五 激励机制

### 5.1 作者激励

- **首发奖励**：原创首发文章额外奖励
- **精品奖励**：被选为「精选文章」奖励

### 5.2 精选文章标准

- 技术深度足够
- 内容独到见解
- 读者反馈良好
- 配图美观规范

## 六 常见问题

### Q: 稿件可以一稿多投吗？

A: 原创文章默认不允许一稿多投。如已在他处发布，请在投稿时说明。

### Q: 已经发布过的文章可以再次投稿吗？

A: 可以，但需要是全新内容或大幅度修改后的版本。

### Q: 投稿后可以撤稿吗？

A: 在正式发布前可以申请撤稿，请通过原投稿渠道说明。

### Q: 如何联系编辑？

A: 邮箱：`shenhuanjie@email.com` | GitHub Issue
