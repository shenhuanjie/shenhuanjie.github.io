Nano Banana 这一类图片生成与编辑工具，最适合处理“给定一张或多张参考图，然后按照明确目标生成新图”的任务。它的玩法不只是单纯文生图，而是把人物、服装、姿势、风格、背景、材质和输出规格组合起来，形成可以复用的图片生产流程。

原文收集了大量热门玩法，覆盖手办化、肖像、自拍、合照、图像融合、虚拟换装、修复增强、背景替换、天气时间变化等方向。下面按照使用场景重新整理，方便后续直接做成自己的提示词库。

## 一、提示词的通用结构

写图片提示词时，不要只写“把图片变好看”。更稳定的写法是把需求拆成几个字段：

```text
主体：要处理的人物、角色、物品或场景
任务：希望图片发生什么变化
保留：必须保持不变的身份、五官、服装、构图或比例
风格：写实、胶片、手办、插画、电影感、产品摄影等
画面：背景、光线、镜头、材质、氛围、细节
限制：不要文字、水印、变形、多余肢体、身份漂移等
输出：横竖构图、网格、近景、全身、高分辨率等
```

这套结构适用于大多数图片编辑任务。尤其是有人脸或产品主体时，“保留什么”和“不要什么”通常比“变成什么”更重要。

## 二、玩法分类速览

### 1. 手办、公仔与收藏品方向

适合把人物、插画角色或宠物转换成实体收藏品效果。

可归纳为几类：

- 3D 展示手办：强调比例、底座、桌面、模型预览和包装盒。
- Funko Pop 公仔：强调大头比例、包装盒、配件陈列和等角展示。
- Q 版钥匙扣：强调软胶材质、描边、钥匙圈和中性背景。
- 针织玩偶：强调毛线纹理、手托近景、暖色环境和手作质感。
- 透明扭蛋胶囊：强调微缩场景、透明外壳、底座颜色和角色标识。

这类提示词的关键是“把数字图像转换成真实可拍摄的物体”。可以多写材质、尺度、摆放环境和拍摄方式。

### 2. 肖像与摄影方向

适合把普通照片转成更强的社交头像、杂志写真或情绪大片。

常见方向包括：

- 黑白艺术肖像：关注脸部轮廓、柔光、颗粒感、留白和情绪。
- iPhone 随手自拍：关注随意构图、轻微虚焦、曝光不完美和现场感。
- 宝丽来合照：关注闪光灯、室内派对感、相纸质感和人物亲密关系。
- 博物馆人像：关注人物与背景作品的空间关系，避免遮挡主体。
- 电影风格肖像：关注低角度、强对比光影、背景色和人物气场。

这类提示词要把“摄影语言”写清楚，比如镜头距离、光源方向、背景深度、胶片颗粒、负空间和构图位置。

### 3. 多图融合与参考图编辑

很多 AI 图片玩法不是单图处理，而是让多张图互相提供信息。

典型任务包括：

- 多图融合：把多个主体放进统一画面，保持光影和透视一致。
- 风格融合：让第一张图保留主体，让第二张图提供色彩、纹理或画风。
- 虚拟换装：保留人物身份和姿势，只替换指定服装。
- AI 化妆：保留人物姿势和五官，把另一张图的妆容迁移过来。
- 对象提取：从人物或场景中提取单个服饰、物品或素材。

这类提示词建议明确写出 `Image1`、`Image2` 分别承担什么角色，避免模型把主体、风格和参考关系混在一起。

## 三、可复用提示词模板

下面是整理后的通用模板，可以按自己的图片替换方括号内容。

### 1. 角色转 3D 展示手办

```text
基于 [Image1] 中的角色生成一张写实产品摄影图。角色被制作成精致的 3D 收藏手办，保持原有五官、发型、服装和标志性姿势。手办放在简洁桌面上，带透明圆形底座，旁边有同系列包装盒和建模预览屏幕。整体光线柔和，材质真实，细节清晰，不添加无关文字和水印。
```

### 2. 电影感人物肖像

```text
将 [Image1] 处理成竖幅电影级肖像。保持人物身份、五官、发型和服装风格一致，使用强方向光和高对比阴影突出面部轮廓。背景为 [背景色或场景]，画面具有胶片颗粒、浅景深和戏剧化氛围。构图干净，避免面部变形、过度磨皮和多余文字。
```

### 3. 自然手机自拍

```text
根据 [Image1] 生成人物在 [地点] 的普通手机自拍。画面像临时拿起手机拍下的一张照片，构图略随意，光线真实，有轻微动态模糊和自然曝光偏差。保持人物面部特征一致，不要商业写真感，不要过度修饰。
```

### 4. 宝丽来合照

```text
把 [Image1] 与 [Image2] 生成为一张室内闪光灯拍摄的宝丽来合照。两人动作自然亲近，表情轻松，照片带轻微模糊、相纸边框和即时成像质感。保持两个人的面部身份一致，背景简单，避免出现额外人物或错误肢体。
```

### 5. 多图融合场景

```text
将 [Image1]、[Image2] 和 [Image3] 中的主体合成到同一个 [场景] 中。所有主体保持清晰可辨，尺寸比例合理，光线方向、透视关系和色彩风格统一。画面自然真实，不要拼贴痕迹，不要让主体相互遮挡关键细节。
```

### 6. 虚拟换装

```text
保持 [Image1] 中人物的身份、姿势、体型比例和面部特征不变，将服装替换为 [Image2] 中的 [服装类型]。新服装需要贴合身体姿态，材质、褶皱、光影和边缘自然。背景保持原样，不改变发型和表情。
```

### 7. 背景替换

```text
将 [Image1] 的背景替换为 [目标背景]，保留主体的人物比例、发型、服装、边缘细节和原始光影关系。新背景需要与主体自然融合，景深合理，色温协调。不要改变主体身份，不要产生边缘毛刺或虚假阴影。
```

### 8. 表情与姿势控制

```text
保持 [Image1] 中人物身份、服装、发型和画面风格一致，将表情调整为 [目标表情]，姿势调整为 [目标姿势]。动作要自然可信，面部结构稳定，手部和肢体比例正确。不要改变背景，不要生成多余人物。
```

### 9. 老照片修复与上色

```text
修复 [Image1] 这张旧照片。去除划痕、污渍、霉斑和褪色问题，恢复人物五官、服装纹理和背景层次。进行自然上色，颜色克制真实，不要过度锐化，不要改变人物身份和原始构图。
```

### 10. 线稿转完整插画

```text
将 [Image1] 的线稿转换成完整彩色插画。保留原始轮廓、构图和角色设计，应用 [目标风格] 的色彩、材质和光影。画面细节丰富，边缘干净，主体突出，不要改变核心造型。
```

## 四、使用时的注意事项

1. 人像任务优先写“保持身份一致”，否则容易脸变。
2. 换装、化妆、风格迁移类任务要明确每张参考图的作用。
3. 如果要求真实感，多写光源、镜头、材质、景深和环境。
4. 如果要求产品感，多写展示角度、底座、包装、干净背景和高分辨率。
5. 如果结果有多余文字或水印，在限制里直接写“不要文字、Logo、水印”。
6. 如果手部、肢体或多人合照容易出错，可以增加“肢体比例正确、手指自然、不要额外人物”。
7. 多步骤编辑建议一次只改一个目标，例如先换背景，再做调色，再做构图扩展。

## 五、适合沉淀成提示词库的字段

后续可以把每个玩法整理成结构化模板：

| 字段 | 示例 |
| --- | --- |
| 玩法名称 | 3D 手办、电影肖像、背景替换 |
| 输入图片 | 单图、双图、多图 |
| 保留内容 | 人脸、姿势、服装、构图、背景 |
| 变化目标 | 风格化、融合、修复、替换、增强 |
| 画面约束 | 无文字、无水印、无额外人物、比例正确 |
| 输出规格 | 头像、竖图、横图、九宫格、产品图 |

这样一来，同一个提示词可以被快速迁移到不同项目里。真正影响出图稳定性的，不是模板写得多复杂，而是输入关系、保留目标和画面限制是否足够清楚。
